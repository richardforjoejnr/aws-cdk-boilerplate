# Phase 0 Spike (throwaway)

Proves browser → Cognito → AWS IoT Core MQTT-over-WSS + Web Speech (ADR-6 / design-review F-6). Results: `../docs/planning/spike-results.md`.

```bash
# 1. Deploy the gated spike stack
cd packages/infrastructure && DEPLOY_GHANA_SPIKE=true STAGE=dev npx cdk deploy dev-ghana-payments-spike --require-approval never

# 2. Generate .env + browser/config.js from stack outputs
./packages/ghana-payments/spike/configure.sh dev

# 3. Headless verification (waits for a publish)
node packages/ghana-payments/spike/node-client.mjs

# 4. Browser soundbox test
cd packages/ghana-payments/spike/browser && python3 -m http.server 8642
# open http://localhost:8642, click "Pair & Connect"

# 5. Trigger the announcement
aws iot-data publish --topic spike/announce --cli-binary-format raw-in-base64-out \
  --payload '{"message":"Payment received, 20 Ghana cedis","payment_id":"pay_demo_1"}'

# Teardown (throwaway — destroy once ADR-6 decision is recorded)
cd packages/infrastructure && DEPLOY_GHANA_SPIKE=true STAGE=dev npx cdk destroy dev-ghana-payments-spike --force
```

Note: `browser/config.js` and `browser/mqtt.min.js` are generated by `configure.sh`; `.env` contains no secrets (identity pool is public by design for unauth flow).
